Data Models (Key Fields)
User
email, password(hashed), name, role(admin|supervisor|employee|contractor), phone, avatar, isActive

Industry
name, code, description, address, contactPerson, contactPhone, status(active|inactive), createdBy

Contractor
name, email, phone, company, industry(ref), employees[ref], status, createdBy

Employee
name, email, phone, department, designation, salary, joiningDate, status(active|inactive), supervisor(ref), industry(ref), contractor(ref), bankDetails, emergencyContact, addedBy

Supervisor
user(ref), industry(ref), employees[ref], department

Attendance
employee(ref), date, status(present|absent|halfDay|leave), checkIn, checkOut, shift(ref), overtime, markedBy

Shift
name, startTime, endTime, breakDuration, isActive

Leave
employee(ref), type(sick|casual|earned|unpaid), startDate, endDate, days, reason, status(pending|approved|rejected), approvedBy, appliedAt

Payroll
employee(ref), month, year, basicSalary, allowances, deductions, overtime, leaveDays, netPay, status(draft|generated|locked), generatedAt, lockedAt

SalaryStructure
designation, basicPercent, hraPercent, daPercent, pfPercent, otherAllowances

Notification
user(ref), title, message, type(info|warning|success|error), isRead, link, createdAt

AuditLog
user(ref), action, module, targetId, changes(before/after), ip, timestamp

Execution Strategy
IMPORTANT

This system has ~50+ backend files and ~40+ frontend files. I will build it in phases, delivering working functionality at each phase. Each phase builds on the previous one.

Phase 1 вЂ” Foundation (this step)
Scaffold all 6 backend services + gateway
Restructure frontend with React Router + sidebar layout
Auth service: registration + login + RBAC + JWT
Reusable frontend components (DataTable, Modal, FormField, etc.)
Phase 2 вЂ” Core CRUD
Industry, Contractor, Employee, Supervisor full CRUD
All 4 list/form pages in frontend
Assignments (employeeв†”supervisor, contractorв†”industry, etc.)
Phase 3 вЂ” Operations
Attendance marking + bulk entry + shift management

Check-in/check-out + overtime calculation
Phase 4 вЂ” Finance

Phase 5 вЂ” Platform
In-app notification system

Phase 6 вЂ” Dashboards
